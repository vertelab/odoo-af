Description

This integration replaces AIS Beslut Om Stod Read v1.
Endpoint: GET /arbetssokande/arenden.
Jira: AFC-734: AIS-Å integration.

This an Experience API for system that handles organisational specific cases. Specifically for Odoo it returns a jobseeker's case information either from AIS-Å or BÄR. More detailed flow documentation is available at the IPF API Portal: Ärende För System v2 2.0.x.

Endpoint:
GET /arbetssokande/arenden.

This integration requires basic authentication. Contact integration team for more information.

Configuration

To configure this integration, you need to be logged in as Super User:

  1. Go to Tekniskt > IPF Settings
  2. Select IPF Case
  3. Enter the following parameters
    • Url: https://ipf-acc.arbetsformedlingen.se
    • Port: 443
    • Client ID: da03472cd17e4ce4bb2d017156db7156
    • Client Secret: B4BC32F21a314Cb9B48877989Cc1e3b8
    • Auth User: AfCrm
    • Auth Password: L65&4v+bQUt&5m&4
    • AF-Environment: T2
    • AF-Systemid: AFCRM
See example below.

Test

To test this integration, you need to:

  1. Go to Tekniskt > Åtgärder >Serveråtgärder
  2. Select IPF Case
  3. The env['af.ipf'].browse() should be set to IPF Planning's current enviroment ID. i.e 3
  4. The personnummer should be set to an jobseeker. i.e 196904180012
  5. Run should return a list with the jobseeker's cases
  6. Return to Jobseeker tab and confirm .
  7. Go to Arbestsökande > Ärende >
  8. See example below
