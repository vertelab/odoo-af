Description

This integration provides information that can form the basis for decisions on support. An endpoint return an enum value of the jobseeker's KROM status (Enum values: Krom, KromEsf, EjKrom). The API is an HTTP proxy against the underlying API of AIS. It is used to request information needed as a basis for decisions on support. More detailed flow documentation is available at the IPF API Portal: AIS BOS Regelverk,

Endpoint:
GET /kromtyp-for-postnummer.

*NB THis integration is configuered to only work in "My administrative area". Meanig it will not be trigered in "Jobseekers" view.

Configuration

To configure this integration, you need to be logged in as Super User:

  1. Go to Tekniskt > IPF Settings
  2. Select IPF AIS BOS
  3. Enter the following parameters
    • Url: https://ipf-acc.arbetsformedlingen.se
    • Port: 443
    • Client ID: da03472cd17e4ce4bb2d017156db7156
    • Client Secret: B4BC32F21a314Cb9B48877989Cc1e3b8

    • AF-Systemid: AFCRM

Test

To test this integration, you need to:

  1. Go to Tekniskt > Åtgärder >Serveråtgärder
  2. Select IPF AIS BOS
  3. The env['af.ipf'].browse() should be set to IPF AIS BOS's current enviroment ID. i.e 3
  4. The postnummer should be set to that of a jobseeker. i.e 34230
  5. Run should return a jobseeker's KROM eligibility based on their postcode.
  6. Return to Jobseeker tab and confirm .
  7. Observe the Rusta och matcha-område field
